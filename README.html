READ ME - Generating accessible documents with Pandoc

This document explains how to use the files in this archive (you should have found it inside AccessibleDocTool-master.zip).

What is Pandoc? How can I use it?

Pandoc is a powerful text file converter. It can convert between formats like DOCX, LaTeX, PDF, EPUB, TXT, MD and much more.

We're interested in its ability to convert from Markdown (Wikipedia article on Markdown), a simple to use formatting system to the following formats:

  • DOCX - regular Microsoft Word files that are easier to navigate with screen readers;
  • PDF - portable files that look crisp at any zoom level for low vision users;
  • EPUB - book publication files that can be read on multiple devices and programs and easily allow users to adapt colors and fonts families and sizes to their needs.

After installing Pandoc and a LaTeX package, you'll be able to write accessible documents in Markdown and convert them by running the .bat file on Windows or the .sh file on Linux or macOS.

What do I install again?

Pandoc and a LaTeX distribution.

Installing Pandoc

Go to Pandoc's installation page and download the installer for your platform. It'll be the first link in the page's main content. There are other installation options you may prefer, if you're a more technical individual.

Installing LaTeX

After installing Pandoc itself, you'll need LaTeX. LaTeX is a powerful typesetting markup language, usually used by academics, but we only need it as an intermediate step for generating PDFs.

If you don't need to generate PDFs, skip this step.

Pandoc recommends the following options depending on your operating system:

  • Windows - MiKTeX;
  • macOS - MacTeX;
  • Linux - install the texlive package.

Since this step is particularly platform dependent further details are outside the scope od this guide.

OK, so what are all these files? Where do I start?

These are the files contained in this archive and what they're for:
File name Description
README.html This file. Hi!
cover.jpg A cover for the EPUB file - replace it with the appropriate image.
title.txt Contains the metadata for the document - title and subtitle, author, etc.
text.md Contains the document itself, formatted in Markdown.
accessible.docx A template Word file with styles appropriate for low vision users.
accessible.css Contains styles that are appropriate for low vision users for EPUB files.
build.bat The script to build the converted files on Window.
build.sh The script to build the converted files on macOS or Linux.
out The folder where the converted files go.

I installed everything, now what?

Run either build.bat or build.sh depending on your operating system.

You should now have a DOCX, an EPUB and a PDF file in the "out" folder. If not, there's something wrong with your installation. Unfortunately, there are too many scenarios, so troubleshooting is outside the scope of this guide.

Writing and converting documents

  1. Fill in the metadata in title.txt;
  2. Get a cover page and rename it to cover.jpg;
  3. Write the actual document, formatted in Markdown in text.md;
  4. Generate fhe output files by running either build.bat or build.sh depending on your operating system.

Your output files should now be in the out folder!
